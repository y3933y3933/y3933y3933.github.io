使用Compound Pattern建立簡易Expand元件

16.11.2023 — React — 4 min read

什麼是 Compound Pattern?

Compound Pattern 是一種 React 的元件設計模式。

Compound指的是組合，意思是由兩個或以上的元件所組合而成的元件，且會有一個主要的元件(通常是parent component)，其它元件如果不在主元件內使用就無效。

常見的例子如：Select、Expand 元件

<Select>
  <Option>選項1</Option>
  <Option>選項2</Option>
<Select>

實作技巧一：使用useContext傳遞state

使用 Expand 元件，我們會需要一個 state 跟更新 state 的 function，用來控制 body 內容是否顯示，這裡命名為expand與toggle，因為需要傳遞給其他子元件使用，可以用useContext來實作。

  1. 透過createContext建立一個context
  2. 使用context.Provider 傳遞state
import React, { createContext, useState } from "react";


const ExpandContext = createContext();


const Expand = ({ children }) => {
  const [expanded, setExpanded] = useState(false);
  const toggle = setExpanded((prevExpanded) => !prevExpanded);
  const value = { expanded, toggle };
  return (
    <ExpandContext.Provider value={value}>{children}</ExpandContext.Provider>
  );
};


const Header = ({ children }) => {
  const { toggle } = useContext(ExpandContext);
  return <div onClick={toggle}>{children}</div>;
};


const Body = ({ children }) => {
  const { expanded } = useContext(ExpandContext);
  return expanded ? children : null;
};


Expand.Header = Header;
Expand.Body = Body;


export default Expand;

實作技巧二：使用useCallback＆useMemo避免重複渲染問題

上述程式碼執行後，會出現以下錯誤：

原因是我們所傳遞的value跟toggle型別是object跟function，React re-render時會認為其refernece不同，導致子元件也會不停的被re-render。

解決辦法為：

  • toggle加上uesCallback，dependencies設為空陣列，確保初始化後toggle就不會再被改變
  • value加上uesMemo，dependencies帶入expanded及toggle，當相依值有改變，value才會跟著變。
const toggle = useCallback(() => {
    setExpanded((prevExpanded) => !prevExpanded);
  }, []);
const value = useMemo(() => ({ expanded, toggle }), [expanded, toggle]);

實用技巧三：透過useRef解決useEffect第一次mount時就會觸發

元件基本功能已完成，但我們還想加上一個onExpand參數，當expand狀態改變時，可以傳入處理的事件。 我們改寫一下Expand，加上onExpand prop跟 uesEffect的處理：

const Expand = ({ children, onExpand = () => {} }) => {
  const [expanded, setExpanded] = useState(false);
  const toggle = useCallback(() => {
    setExpanded((prevExpanded) => !prevExpanded);
  }, []);
  const value = useMemo(() => ({ expanded, toggle }), [expanded, toggle]);


  useEffect(() => {
    onExpand(expanded);
  }, [expanded]);
  return (
    <ExpandContext.Provider value={value}>{children}</ExpandContext.Provider>
  );
};

但這裡會遇到一個問題，我們希望在元件mount時不要觸發onExpand，但useEffect預設會在mount的時候被觸發。

解決辦法為透過useRef加入一個flag判斷，useRef是可以讓元件記住某些資訊，可以re-render時該值不會被改變，我們可以用它來紀錄是否為mount的時機點：

const Expand = ({ children, onExpand = () => {} }) => {
  const [expanded, setExpanded] = useState(false);
  const toggle = useCallback(() => {
    setExpanded((prevExpanded) => !prevExpanded);
  }, []);
  const value = useMemo(() => ({ expanded, toggle }), [expanded, toggle]);


  const isMounted = useRef(true);


  useEffect(() => {
    if (!isMounted.current) {
      onExpand(expanded);
    }
    isMounted.current = false;
  }, [expanded]);
  return (
    <ExpandContext.Provider value={value}>{children}</ExpandContext.Provider>
  );
};

Summary

  • Compound Pattern 是一種元件設計模式，是指由一個主元件跟其他元件組合而成的元件。
  • Compound Pattern 常搭配useContext
